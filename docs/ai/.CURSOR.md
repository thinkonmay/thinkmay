2. **Mobile implementation** → `docs/ai/mobile/specs/` + `docs/ai/mobile/TASK.md`

## When editing mobile code

Work in the `mobile/` submodule (`lib/`), but update specs and task log in **`docs/ai/mobile/`** in the parent repo — not inside the submodule.
